# KHAOS 跨平台指标引擎 Spec

## Why
当前 `KHAOS.pine` 已经暴露出明显边界：Pine Script 难以无损承载 PI-KAN 的训练语义，且同花顺等平台各自存在语法、性能、运行环境限制。如果继续为每个平台单独手写一版指标，不仅会持续丢失训练语义，还会放大性能与维护成本。

## What Changes
- 新增一套“跨平台指标引擎”方案评估与落地规格，用统一核心替代“每个平台各写一版逻辑”。
- **BREAKING**: 后续不再把 Pine / 同花顺脚本视为权威信号源，而是视为可视化适配层或轻量表达层。
- 明确双轨混合方案为首选路线：保留本地高性能核心能力，同时预留服务端统一信号分发能力。
- 明确 Python、Rust、Pine、同花顺公式语言在体系中的职责边界与适用范围。
- 输出一份面向开发的跨平台架构文档，说明可行路径、性能约束、适配策略和后续实施顺序。

## Impact
- Affected specs:
  - 指标跨平台运行架构
  - 训练语义到运行时的映射方式
  - 多市场多终端的信号交付方式
  - 高性能计算核心与前端适配层的职责边界
- Affected code:
  - `Finance/02_核心代码/源代码/khaos/Pine Script代码/KHAOS.pine`
  - `Finance/02_核心代码/源代码/khaos/核心引擎/physics.py`
  - `Finance/02_核心代码/源代码/khaos/数据处理/data_loader.py`
  - `Finance/02_核心代码/源代码/khaos/回测模块/symbolic_extractor.py`
  - `Finance/04_项目文档/02_开发文档/02_技术文档/`

## ADDED Requirements
### Requirement: 跨平台指标核心方案
系统 SHALL 提供一套统一的跨平台指标核心方案，用于支撑 TradingView、同花顺、未来网页端/桌面端以及服务端调用，而不是依赖各平台脚本语言分别重写核心逻辑。

#### Scenario: 统一核心定义
- **WHEN** 开发者为不同市场或不同终端接入 KHAOS 指标
- **THEN** 应先复用统一的核心特征、状态定义、阈值口径与输出枚举，再在平台层做最小适配

### Requirement: 双轨混合运行架构
系统 SHALL 采用“双轨混合”作为推荐落地形态：一条轨道负责本地高性能计算核心，另一条轨道负责服务端统一信号分发与多终端接入。

#### Scenario: 多平台接入
- **WHEN** 指标需要同时支持 Pine、同花顺或其他客户端
- **THEN** 应允许平台内脚本只承担轻量显示或有限规则表达，权威信号计算由统一核心或服务端输出

### Requirement: 语言职责分层
系统 SHALL 明确 Python、Rust 与平台脚本语言各自职责，避免再把训练核心、精确推理和图表脚本混写在同一层。

#### Scenario: 选择实现语言
- **WHEN** 团队评估 Python 或 Rust 是否适合作为跨平台内核
- **THEN** 规格必须清晰说明 Python 更适合作为参考实现与编排层，Rust 更适合作为高性能跨平台核心与多语言绑定层

### Requirement: 训练语义保真优先
系统 SHALL 把“训练语义保真”作为高于“平台内原地实现”的目标，明确哪些平台适合做权威信号输出，哪些平台只能做近似可视化。

#### Scenario: 平台能力不足
- **WHEN** Pine Script、同花顺公式语言等平台无法无损承载 PI-KAN 训练语义
- **THEN** 方案必须优先推荐核心外置、结果分发、轻量适配，而不是继续在平台脚本里堆叠复杂近似逻辑

## MODIFIED Requirements
### Requirement: Pine 指标的系统定位
现有 Pine 指标必须从“权威信号实现”调整为“前端图表表达层”或“轻量级近似适配层”。它可以展示状态、接收统一结果或表达简化规则，但不得再被视为训练主语义的唯一真实实现。

### Requirement: 规则提取与符号映射
现有 `symbolic_extractor.py` 和阈值/规则提取能力必须从“直接替代全部运行时”调整为“服务跨平台适配的中间层能力”，用于构建轻量公式、校验平台近似和支撑核心规范，而不是单独承担完整跨平台落地。

## REMOVED Requirements
### Requirement: 每个平台独立重写完整指标核心
**Reason**: 该方式会持续造成语义漂移、性能退化和维护成本膨胀，且已被 Pine 映射问题验证不可持续。
**Migration**: 迁移到“统一核心规范 + 高性能核心实现 + 平台适配层 + 服务端分发能力”的新架构。
