# 重新设计相变探测训练目标 Spec

## 为什么 (Why)
之前的方案试图人为设定硬性阈值，这限制了神经网络对真实连续分布的拟合能力。而仅预测“波动率比率”虽然解决了硬阈值问题，却丢失了“波段反转”的方向性信号。为了在不限制网络发挥的前提下，**精准且同时**捕捉“波动率暴增”和“波段趋势反转”，我们应当引入**多任务连续目标 (Multi-Task Continuous Targets)**。通过将物理学中的“能量激增”和数学上的“均值回归交叉相关性”转化为两个独立的连续回归目标，模型可以自然地从数据中学习到这两种截然不同的相变模式，彻底告别人为设定的条件限制。

## 做了哪些更改 (What Changes)
- **BREAKING**: 重构 `data_loader.py` 中的 `self.targets`，将其维度扩展为 2 维 (`[N, 2]`)：
  - **Target 0 (波动率扩张率 - Volatility Expansion)**: `np.log(forward_vol / (sigma + 1e-8))`。该值衡量未来实际波动率相对于历史波动率的放大倍数。$>0$ 代表波动率暴增（相变爆发），$<0$ 代表死水期。
  - **Target 1 (均值回归强度 - Mean Reversion Strength)**: `-((close - ema) / sigma) * (forward_return / sigma)`。该值是“当前均线偏离度”与“未来收益率”的负向交叉项。当价格严重偏离均线（极值）且未来发生强烈反向运动（反转）时，该值为巨大的正数；若顺势继续发散，则为负数。这是一个完美定义反转强度的连续无界变量。
- **BREAKING**: 修改 `train.py`，将 `KHAOS_KAN` 的 `output_dim` 实例化参数从 1 改为 2。损失函数保持 `MSELoss`。
- **自适应样本权重**: 修改 `sample_weights` 计算，使其与两个目标值的极值挂钩（例如 `weight = 1.0 + np.maximum(0, target_0) + np.maximum(0, target_1)`），让模型自发地关注那些爆发或反转极其剧烈的样本。
- **物理正则化适配**: 在 `loss.py` 中，解包多维预测值。
  - `pred[..., 0]` (波动率预期): 当物理特征（如高 Entropy、高 MLE）预示系统混沌爆发时，若预测 $<0$，施加惩罚。
  - `pred[..., 1]` (反转预期): 当处于均线极值且残差突变时，若预测 $<0$，施加惩罚。

## 新增需求 (ADDED Requirements)
### 需求：多维连续相变目标预测
系统必须能够同时预测未来市场的波动率扩张级别以及均值回归（反转）的动能强度。

#### 场景：模型学习爆发与反转
- **当** 数据加载器遍历历史序列时
- **则** 生成一个 2D 的连续 Target 向量。Target 0 负责指导模型识别无方向的剧烈波动（Breakout），Target 1 负责指导模型识别波段顶底的剧烈反转（Reversal）。两者均无硬性阈值切分。

## 修改的需求 (MODIFIED Requirements)
### 需求：模型结构与物理损失函数 (Physics Loss)
模型必须输出 2 维向量。`loss.py` 中的 `PhysicsLoss` 必须将正则化条件分别映射到对应的输出维度上，确保物理先验能够准确指导特定的相变模式。

## 移除的需求 (REMOVED Requirements)
### 需求：方向性连续收益率目标、人为硬阈值
**原因**：单一的收益率目标导致模型退化为动量追踪器，硬阈值会破坏数据连续性。
**迁移**：完全被连续的“波动率扩张率”和“均值回归强度”所替代。
