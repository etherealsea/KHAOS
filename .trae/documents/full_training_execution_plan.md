# 全量多品种多时间周期训练执行计划

## 1. 目标
在最新的双维度（波动率扩张率、均值回归强度）连续目标框架下，使用全量多品种（加密货币、大宗商品、股指）、多时间周期（5m, 15m, 1h, 4h, 1d）数据进行完整的模型训练。

## 2. 执行步骤

### 步骤 1：数据准备与环境检查
- **操作**：检查 `Finance/01_数据中心/03_研究数据/research_processed/training_ready/` 目录下是否包含所有规定的品种和周期数据。
- **验证**：排除所有外汇（如 EURUSD），确保包含 BTC, ETH, GOLD, WTI, SPX, NDX 等核心资产。

### 步骤 2：配置并启动全量训练
- **操作**：修改并运行 `train.py`。
- **参数设定**：
  - 关闭 `--test_mode`。
  - 设定合理的 `epochs`（例如 5-10，视收敛情况而定）。
  - 保持 `batch_size=256` 或根据显存动态调整。
  - 确保启用 AMP 混合精度以及前向/反向传播的 NaN/Inf 检查。

### 步骤 3：模型监控与保存
- **操作**：监控训练过程中的 Loss 下降情况（包含主 MSE Loss 和 P3/P4/P6/P7 物理正则化 Loss）。
- **预期结果**：成功生成 `khaos_kan_model_final.pth` 权重文件，且不出现维度崩溃。

### 步骤 4：提取双维度规则与注意力分布
- **操作**：运行更新后的 `symbolic_extractor.py`。
- **预期结果**：
  - 提取针对“波动率扩张”的物理阈值（如 Entropy, MLE 的触发点）。
  - 提取针对“均值回归”的物理阈值（如 EKF_Res, EMA_Div 的触发点）。
  - 输出模型对时序上下文（20 个窗口）的注意力权重分配情况。

### 步骤 5：更新开发日志
- **操作**：将完整的训练结果、Loss 收敛情况以及最新提取出的数学公式/交易规则写入当日的开发日志（`2026-03-24.md`）。
