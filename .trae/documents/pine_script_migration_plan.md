# PI-KAN 模型至 Pine Script 移植计划

## 1. 目标与背景
基于全量数据训练出来的多任务目标模型，我们需要将 `symbolic_extractor.py` 最新提取出来的非线性物理阈值和时序注意力规则无损移植到 Pine Script v6 中，以指导实盘交易。

## 2. 核心更新点分析

根据最新的提取日志（2026-03-24）：
- **趋势基准**：`Hurst > -0.90` 
  - *解释*：这里的阈值是经过 RevIN（类似 Z-Score）归一化后的值。Z-Score 为负数（如 -0.90）代表该值低于历史平均值近一个标准差。这意味着网络学习到，即使 Hurst 指数稍微低于历史均值（系统并非处于极强单边趋势），只要不极度陷入均值回归（如 < -2.0），系统就具备爆发的**环境土壤**。这大大降低了捕捉突破行情的门槛。
- **混沌突破相变 (Target 0 波动率扩张)**：
  - `Volatility > -0.49`
    - *解释*：同理，这里的 Volatility 是 Z-Score 后的值。`> -0.49` 意味着波动率不需要突破历史新高，只要它从极度死水期（如 Z-Score < -1.5）开始复苏，回升到接近历史均值水平（-0.49），此时配合高熵或高MLE，网络就认为相变已经开始。这说明模型学会了**“在左侧捕捉波动率抬升”**，而不是等波动率已经飙升到 +2.0 才去追高（右侧）。
  - `Entropy > 0.54`
  - `MLE > 0.17`
- **极值反转相变 (Target 1 均值回归)**：
  - `abs(EKF_Res) > 1.00`
  - `abs(EMA_Div) > 1.00`
- **注意力分布**：
  - 局部触发锚点变为 `t-3`（原为 t-5/t-10），远期记忆占比 59.3%。这表明策略应该**重点扫描最近 3 根 K 线**是否有异动信号（Trigger Anchor）。

## 3. 移植实施步骤

### 步骤 1：更新 Pine Script 输入参数默认值
- **操作**：修改 `KHAOS.pine` 的 `input.float` 默认值，对齐上述提取出的最新物理阈值。
- **具体修改**：
  - `thres_hurst` 从 `-0.13` 更新为 `-0.90`
  - `thres_mle` 从 `1.5` 更新为 `0.17`
  - `thres_entropy` 从 `1.5` 更新为 `0.54`
  - `thres_vol` 从 `1.5` 更新为 `-0.49`
  - `thres_res` 从 `2.0` 更新为 `1.00`
  - `thres_div` 从 `2.0` 更新为 `1.00`
  - `anchor_len` 从 `5` 更新为 `3`（基于 t-3 的局部触发锚点）。

### 步骤 2：优化状态机逻辑 (不损失效果的重构)
- **操作**：由于模型现在是**多任务双输出**，突破和回归是正交解耦的。当前的 Pine Script 中存在对 `raw_breakout` 的过度严格限制。
- **具体修改**：
  - **爆发条件 (Breakout)**：根据提取规则，`Volatility > -0.49` 且 (`MLE > 0.17` 或 `Entropy > 0.54`) 即为强爆发状态。
  - **反转条件 (Reversion)**：`abs(EKF_Res) > 1.0` 且 `abs(EMA_Div) > 1.0` 即触发。
  - 确保使用 `ta.highest(xxx, 3)` (即 `anchor_len=3`) 的方式实现“过去 3 根 K 线内出现过触发信号即可”，以完美复刻 Attention 机制的 `t-3` 锚点特性。

### 步骤 3：验证与保存
- **操作**：仔细检查 Pine Script v6 语法规范，确保不引入作用域（scope）或变量遮蔽（shadowing）的编译错误。
- **预期结果**：更新后的 `KHAOS.pine` 能够直接复制到 TradingView 中运行，无缝继承本次大规模训练的 AI 智慧。
