# PI-KAN 物理规律训练重构计划

## 1. 目标定义

本次重构的目标不是继续让模型学习“价格偏离后未来几根 K 线怎么走”的近似代理，而是让 PI-KAN 更稳定地学习以下两类更接近物理相变本质的规律：

- **能量释放规律**：系统何时从压缩、脆弱、分歧累积状态，转入可交易级别的波动率扩张与相变爆发。
- **失衡回归规律**：系统何时从趋势一致性走向局部失衡，并在随后进入有效回调甚至大级别反转，而不是普通噪声回踩。

## 2. 当前训练为何学不到更深层规律

结合当前实现，主要问题不在“模型完全没有时序能力”，而在“监督方式和训练组织正在把模型拉回到短期价格代理”：

- [data_loader.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/数据处理/data_loader.py#L22-L53) 的回归目标仍主要围绕当前失衡与短期未来路径构造，能够识别“已经偏离后是否回踩”，但不擅长学习“熵先降后升、趋势共识瓦解、临界慢化”等更高阶演化结构。
- [loss.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/模型训练/loss.py#L42-L68) 的物理约束仍以静态阈值型惩罚为主，本质上更像“纠偏器”，不是“鼓励网络发现潜在状态演化”的监督。
- [kan.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/模型定义/kan.py#L197-L224) 虽然具备注意力残差与 KAN 非线性映射，但最终仍只取最后一个时刻的序列表征；如果训练标签本身不显式奖励“先降后升”“压缩后爆发”这类形态，模型就更容易学成末端状态分类器。
- [extract_rules.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/模型训练/extract_rules.py#L60-L69) 仅用单文件探针提取规则，会把跨资产、跨周期才稳定存在的规律弱化掉。

## 3. 重构原则

- **原则 A：监督先对准“状态转移”，再让 PI-KAN 学映射**  
  标签必须描述“从何种物理背景转入何种结果”，而不是只描述未来收益本身。

- **原则 B：统一 20 周期主窗口，但允许在窗口内部学习动态形状**  
  局部主窗口继续围绕 20，不再增加混乱周期；但在 20 周期内部显式保留水平、斜率、曲率、压缩/扩张、先降后升等演化信息。

- **原则 C：KAN 负责可解释非线性映射，注意力负责时序上下文整合**  
  不把 KAN 退化成普通 MLP 替代品，也不把时序结构全靠人工规则预先写死。

- **原则 D：训练必须让“难负样本”进入核心监督**  
  重点处理“看起来像极值但并未反转”“看起来熵高但并未爆发”的样本，否则模型会继续学习表层偏离。

- **原则 E：验证必须跨资产、跨周期、跨事件类型**  
  不能只看总体 loss，必须验证网络是否真的抓到可复现的物理 motif。

## 4. 计划中的训练设计

### 阶段 1：重构监督目标，从点预测升级为“事件-过程标签”

目标：让两个输出都直接对应可解释的物理过程，而不是价格代理。

- **爆发核标签**
  - 用未来窗口内的波动扩张、区间突破、残差放大、熵上升、MLE 走强等共同定义“相变爆发强度”。
  - 输出不再只盯 `forward_vol / sigma`，而是形成“是否进入能量释放阶段”的连续分数。

- **回归核标签**
  - 用“当前失衡程度 + 未来反向推进幅度 + 持续时间 + 是否打破原惯性”共同定义“有效回归/反转强度”。
  - 显式区分三类样本：  
    1. 普通回踩  
    2. 可交易级别回归  
    3. 大级别反转
  - 标签设计优先围绕 EKF 公允轨道偏离、EMA20 偏离、局部熵结构、Hurst 背景、MLE 背景的联合结果，而不是单一价格偏离。

- **新增辅助标签**
  - 增加“状态转移辅助任务”，例如：
    - 压缩 -> 爆发
    - 趋势一致 -> 分歧升高
    - 极端失衡 -> 有效回归
  - 这些辅助任务不直接替代主任务，而是作为中间监督，帮助模型学习你关心的潜在规律。

### 阶段 2：重构输入表达，让模型看到物理量的演化形状

目标：让模型不仅看到 20 周期内每根 K 的物理值，还能看到这些物理值怎样变化。

- 在 [physics.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/核心引擎/physics.py) 中保留现有 8 个核心物理量，但为关键变量增加派生动态通道：
  - 水平值
  - 一阶变化率
  - 二阶变化率
  - 局部压缩/扩张度
  - 局部标准化位置
- 重点对 `Entropy`、`MLE`、`EKF_Res`、`EKF_Vel`、`EMA_Div` 增加“形状信息”，让“先降后升”“持续钝化”“突然发散”这类模式可被直接感知。
- 所有派生量仍围绕同一个 20 周期主窗口构造，避免再次引入周期口径混乱。

### 阶段 3：重构模型头部，让共享时序表示与可解释输出解耦

目标：保留当前架构优势，但让不同任务学到不同物理子空间。

- 共享部分继续保留：
  - RevIN 输入归一化
  - 注意力残差块
  - KAN 非线性映射
- 结构上改为：
  - **共享时序编码器**
  - **爆发核 KAN 头**
  - **回归核 KAN 头**
  - **辅助状态转移头**
- 对回归核增加显式的“状态摘要向量”，避免它只盯住最后一根 K。
- 若当前 `x[:, -1, :]` 汇聚仍限制时序表达，则改为“注意力汇聚 / 可学习池化 + 最后一时刻残差”的组合，而不是只保留纯末端状态。

### 阶段 4：重构损失函数，从静态阈值惩罚升级为多目标监督

目标：减少“手工阈值推着模型走”，增强“模型自己学到结构”的空间。

- 主损失改为：
  - 连续回归损失
  - 排序损失（让真正强事件分数高于伪事件）
  - 难负样本对比损失
- 物理约束改为两层：
  - **弱先验一致性**：防止明显违背物理直觉
  - **状态转移一致性**：约束相邻阶段的预测排序与逻辑
- 对“伪极值未反转”“高熵未爆发”“趋势延续误判为回归”建立专门 hard negative 惩罚。
- 保留现有可解释性，但减少对固定阈值如 `Entropy > 常数` 的过度依赖。

### 阶段 5：重构采样与训练组织，让模型真正看到稀有相变样本

目标：解决“极端事件稀少，模型被大量平庸样本淹没”的问题。

- 在 [data_loader.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/数据处理/data_loader.py) 中新增事件分层标记：
  - 平稳样本
  - 爆发候选样本
  - 回归候选样本
  - 难负样本
- 在 [train.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/模型训练/train.py) 中引入分层采样或加权批次策略，避免训练被大量无事发生的片段主导。
- 训练顺序建议采用两阶段：
  - 第一阶段：学习状态识别与稳定表征
  - 第二阶段：增强强事件区分与 hard negative 抑制
- 保留真实全量数据，不使用模拟数据，也不做未经许可的截断采样。

### 阶段 6：重构验证体系，判断模型是否真的学到物理规律

目标：验证“学到规律”而不仅是“loss 下降”。

- 在训练脚本中新增以下验证输出：
  - 爆发核：高分样本的未来波动扩张分布
  - 回归核：高分样本的未来反向推进分布
  - 难负样本误报率
  - 不同资产、不同周期上的分桶表现
- 在 [extract_rules.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/模型训练/extract_rules.py) 与 [symbolic_extractor.py](file:///d:/《基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究》/Finance/02_核心代码/源代码/khaos/回测模块/symbolic_extractor.py) 中升级为：
  - 多资产多周期抽样解释
  - 爆发核 / 回归核分头解释
  - 关键 motif 检查，例如熵先降后升、残差持续积累后释放、Hurst 背景切换
- 验证成功的标准不再只是“提取出几个阈值”，而是：
  - 不同资产上能重复出现相近的物理结构
  - 规则与真实未来事件方向一致
  - Pine 映射后仍保留同一语义

## 6. 执行顺序

1. 先重写标签体系与事件分层。
2. 再扩展物理特征表达。
3. 然后改模型头部与时序汇聚。
4. 接着重写损失与采样策略。
5. 再重构训练验证与规则提取。
6. 最后执行完整训练、提取规律、输出训练报告，并视结果再决定 Pine 语义映射是否同步调整。

## 7. 交付标准

计划执行完成后，应至少满足以下结果：

- 爆发核不再只是“高波动率代理预测器”，而能稳定识别能量释放前后的物理状态。
- 回归核不再只是“价格偏离回踩代理预测器”，而能更准确地区分普通回踩、有效回归、大级别反转。
- 提取出的规则不再主要停留在“某个变量大于负值”这一层，而能够部分呈现多变量演化结构。
- 多资产多周期上出现可重复的解释结果，而不是只在单一文件中成立。
- 后续映射到 Pine 时，超买/超卖与爆发语义将来自训练出的真实状态核，而不是临时拼接阈值。

## 8. 风险与控制

- **风险 1：标签过于复杂导致训练不稳定**  
  控制方式：先保留双主任务，再逐步增加辅助任务，不一次性堆满全部监督。

- **风险 2：特征维度扩张后 KAN 训练变脆弱**  
  控制方式：先做有限扩维，优先增加动态派生量而不是盲目扩大输入规模。

- **风险 3：规则解释复杂度上升**  
  控制方式：坚持“共享编码器 + 可解释 KAN 头”结构，避免端到端黑箱化。

- **风险 4：训练时间显著增加**  
  控制方式：先建立小规模验证流程验证标签方向正确，再执行完整多资产多周期训练。

## 9. 本计划的核心判断

PI-KAN 当前并不是“没有能力”学习你提出的潜在规律，而是“训练信号还没有把模型推向这些规律”。  
这次重构的关键，不是简单继续调超参数，而是把训练从“未来价格代理学习”升级为“物理状态转移学习”。
