# 毕设进展报告

**题目**：基于物理信息增强神经网络（PI-KAN）的金融时间序列相变探测研究\
**日期**：2026-04-01

## 一、项目总体目标

本课题围绕金融时间序列中的“相变”现象展开研究。本文所指“相变”，是指市场状态由一种结构区间向另一种结构区间发生切换的过程，典型表现包括波动压缩后的释放，以及价格偏离均衡后的回归。

本阶段重点识别两类核心信号：

- 爆发信号（Breakout）：指价格脱离原有震荡区间、向外快速扩张的时刻。
- 回归信号（Reversion）：指价格在偏离之后重新回到均衡状态的时刻。

因此，本课题的目标不在于短期价格点位预测，也不以单一指标提升为唯一导向，而是构建一套能够描述市场状态切换机制的模型框架，并据此识别两类信号的周期适配关系。

## 二、关键术语界定

为统一全文表述，现对报告中涉及的关键术语界定如下：

| 术语                                         | 定义                                 | 在本项目中的作用                     |
| ------------------------------------------ | ---------------------------------- | ---------------------------- |
| Hurst 指数（Hurst Exponent）                   | 用于刻画时间序列更偏向趋势延续还是均值回归的统计量。         | 用于辅助判断市场当前更接近趋势主导还是回归主导状态。   |
| 扩展卡尔曼滤波（Extended Kalman Filter, EKF）       | 适用于非线性系统的状态估计方法，可在噪声条件下提取较平滑的隐含状态。 | 用于生成 EKF 状态与残差特征，辅助刻画市场潜在状态。 |
| 熵（Entropy）                                 | 用于衡量系统无序程度与不确定性的指标。                | 用于描述市场结构是否趋于混乱，以及状态切换是否明显。   |
| 极大似然估计（Maximum Likelihood Estimation, MLE） | 通过最大化观测数据出现概率进行参数估计的统计方法。          | 用于提取更稳健的概率型与分布型特征。           |
| 压缩（Compression）                            | 指价格或波动处于收敛、压缩并可能向外释放的状态。           | 用于识别爆发信号形成前的酝酿阶段。            |

上述术语共同构成模型对市场状态转换的描述框架，也是后续训练、验证与结果解释的基础。

## 三、项目总体进展

### 1. 研究主线已经从“价格预测”转向“状态识别”

与课题早期相比，当前项目的研究主线已明显收束。研究目标已由短期收益率预测转向市场状态切换识别，并围绕爆发信号（Breakout）与回归信号（Reversion）两条主线组织模型训练、特征设计与实验评价。

当前使用的输入特征主要包括 Hurst 指数、波动率、EKF 状态与残差、熵、极大似然估计相关特征、价格动量、均线偏离度以及压缩状态等。这些特征与注意力残差机制、局部确认分支和 KAN 解释头结合后，构成了当前 PI-KAN / KHAOS 的研究框架。

从阶段成果看，项目已不再停留于“物理指标 + 神经网络”的概念性组合，而是形成了相对统一的任务语义：爆发信号用于识别结构释放与突破，回归信号用于识别失衡后的修复与回归。这一语义框架已可作为论文方法与实验部分的主线支撑。

### 2. 训练主线已完成多轮关键迭代

3 月中下旬以来，训练主线经历了多次迭代，整体演进方向较为清晰：

- `iter4`：引入事件与伪信号之间的边际约束，并使用全局 epoch 选模，避免模型被单个样本的局部高分误导。
- `iter5`：开始直接输出准确率（Accuracy）、精确率（Precision）、召回率（Recall）、F1、事件命中率和伪信号率，使评价方式从“只看相关性”转向“可直接反映分类性能”。
- `iter6`：继续优化早停、非对称约束和双核评价方式，属于边际优化阶段。
- `iter7`：尝试通过收紧事件定义提升标签纯度，但结果显示回归信号（Reversion）正例明显塌缩，表明标签纯度提升与可训练性之间存在权衡。
- `iter8`：正式将观察窗口调整为 horizon=10，并修正回归路径评价中的尺度不一致问题，使回归信号重新恢复可训练性。
- `iter9`：在保持 horizon=10 的前提下，把训练上限从 10 轮提高到 16 轮，并完成关键验证。

综合来看，项目当前已由探索性试错阶段进入训练主线基本成型、实验结论逐步稳定的阶段。

### 3. iter9 的关键结论：最优点明确出现在 Epoch 15

根据现有开发日志，`iter9` 的综合最优点可以明确界定为 **Epoch 15**。该结论可由全训练日志直接对应验证。

从日志可见：

- `Epoch 15` 的验证摘要为：爆发信号相关性约 `0.4188`，回归信号相关性约 `0.4163`；
- `Epoch 15` 的 F1 约为：爆发信号 `0.5622`，回归信号 `0.5400`；
- `Epoch 16` 虽然爆发信号相关性略高到约 `0.4199`，但爆发信号 F1 回落到约 `0.5592`，回归信号相关性也回落到约 `0.4141`；
- 综合考虑指标表现与双核平衡性，`Epoch 15` 仍可作为本轮训练的最优点。

这一结果的阶段性意义主要体现在以下四个方面。

第一，**训练上限从 10 轮提高到 16 轮是成立的。**\
`iter9` 的最优点出现在第 15 轮，说明此前将训练上限控制在 10 轮以内偏于保守；在当前任务设定下，延长训练轮次能够带来实际收益，并有助于模型进入更稳定的收敛区间。

第二，**回归信号（Reversion）在多周期上更稳定。**\
从 `iter9` 的整体训练表现看，回归信号并非仅在单一周期上有效，而是在不同时间尺度上均能维持相对稳定的相关性与识别质量，表明其更接近跨周期的结构性信息。

第三，**爆发信号（Breakout）更偏向短中周期。**\
与回归信号相比，爆发信号对局部动量变化、压缩释放和短时结构突变更为敏感，因此在 15 分钟与 60 分钟等短中周期上具有更高适配性；在更长周期上，其结构优势弱于回归信号。

第四，**该轮优化并非单纯追求指标数值提升，而是用于确认两类信号各自的周期定位。**\
`iter9` 的主要研究价值，在于进一步明确：回归信号（Reversion）更适合作为多周期、结构型的判断主线；爆发信号（Breakout）更适合作为短中周期、触发型的识别主线。该结论对后续论文论证与工程实现均具有直接意义。

### 4. 最新 A 股适配结果：iterA1（若内部口径称为 iter9a1，则对应本轮结果）

仓库中的正式名称为 `iterA1`，对应的即为本轮 **A 股适配训练结果**。

该轮实验的目的，是检验当前双核状态识别框架在 A 股市场环境下是否仍具备稳定性与可迁移性。

根据已有实验报告 `KHAOS_A股_iterA1_Training_Report`，当前关键结果如下：

- 爆发信号相关性（breakout\_corr）约为 `0.2830`
- 回归信号相关性（reversion\_corr）约为 `0.3560`
- 爆发信号 F1 约为 `0.4828`
- 回归信号 F1 约为 `0.5039`

从总体结果看，回归信号（Reversion）整体仍强于爆发信号（Breakout），并在相关性与 F1 两个层面表现出更好的均衡性。这与 `iter9` 所呈现的结论一致，即回归信号更适合承担跨周期的结构判断任务。

实验报告中的分周期结果如下：

| 周期    | 爆发信号相关性  | 回归信号相关性  | 爆发信号 F1  | 回归信号 F1  |
| ----- | -------- | -------- | -------- | -------- |
| `15m` | `0.3027` | `0.3453` | `0.4861` | `0.5013` |
| `60m` | `0.2333` | `0.3744` | `0.4918` | `0.5117` |
| `1d`  | `0.1650` | `0.4197` | `0.5064` | `0.5124` |

基于上述分周期结果，可归纳出以下三点：

- 在 `15m` 周期上，爆发信号对短线节奏变化响应更敏感，更适合承担突破捕捉与局部触发任务。
- 在 `60m` 周期上，爆发信号与回归信号之间的配合关系更为均衡，是当前短中周期衔接最自然的观察尺度。
- 在 `1d` 周期上，爆发信号相关性下降至 `0.1650`，而回归信号相关性提升至 `0.4197`，表明日线周期更适合作为结构确认层，回归信号优势更为明显。

需要强调的是，此处结论并非依据单一指标排序得出，而是综合相关性、F1 以及信号功能定位后形成。总体而言，A 股适配结果继续支持如下判断：**回归信号（Reversion）偏向多周期稳定识别，爆发信号（Breakout）偏向短中周期快速触发。**

### 5. 应用表达层已有进展，当前重点转向规则化表达与验证接口建设

除训练之外，项目在终端表达层上也取得了一定进展，例如同花顺方向已形成能够展示主线信号的副图表达版本。该方向的定位，是将训练结果转换为可验证的图形化表达，而非要求终端公式完整复刻神经网络本体。

当前阶段的重点，不仅在于模型训练结果本身，还在于将模型结论转化为可验证的应用接口。从这一定位出发，同花顺方向更适合作为表达层与验证层，而不作为复杂模型本体的最终承载平台。

## 四、平台路线与实现约束

平台路线之所以尚未最终确定，主要原因在于 **TradingView 的 Pine Script 在表达复杂模型时存在明显的效能限制**。

Pine Script 更适合轻量级规则脚本，例如均线交叉、简单条件过滤、基础指标叠加和常规告警逻辑；但对于本项目涉及的复杂神经网络、多因子联合建模、非线性状态估计和高维状态推理任务，Pine Script 并不适合作为最终承载平台。

因此，Pine Script 适用于轻量规则表达，不适用于复杂神经网络推理或高维状态推理。当前平台路线仍需持续观察，其核心约束在于平台能力与模型复杂度之间存在明显错位。

目前正在等待 FastBull 平台后续是否支持使用 Python 等语言实现指标功能。如果 FastBull 后续开放这一能力，那么本项目中的特征计算、状态估计和模型输出逻辑才有可能以更完整的方式迁移过去。

在 FastBull 相关能力尚未明确之前，平台路线仍处于**观察和等待阶段**。现阶段不宜对最终承载平台作出过早判断，应继续同步推进研究主线固化与平台能力跟踪两项工作。

## 五、目前存在的主要问题

虽然项目已形成相对明确的研究主线，但当前仍存在几类需要继续解决的问题。

第一，终端公式类实现本质上仍是规则蒸馏后的近似表达，难以完整保留 PI-KAN 模型中的非线性组合关系和双核评分空间。因此，表达层可以用于展示和验证，但不能直接等同于原始模型能力。

第二，A 股结果虽然已经形成阶段性成果，但误报率仍有继续优化空间，说明参数、阈值和竞争规则还需要进一步回放和校准。

第三，平台路线目前尚未完全确定。TradingView 的 Pine Script 不适合承载复杂模型，而 FastBull 对 Python 等语言的支持仍未明确，因此平台问题尚未进入最终定稿阶段。

第四，论文材料虽然已经较多，但要真正形成毕业论文正文，仍需要把“方法设计”“训练演进”“A 股验证”“平台约束与应用表达”整合为统一叙述，否则后续写作会显得材料分散。

## 七、阶段性结论

总体来看，本项目已经完成了从“研究方向探索”向“主线方法成型”的关键过渡。

当前阶段的主要结论可概括为四点：

- `iter9` 的综合最优点已经可以明确写为 **Epoch 15**，这说明训练上限从 10 轮提高到 16 轮是成立的。
- 回归信号（Reversion）在多周期上更稳定，适合作为结构型判断主线。
- 爆发信号（Breakout）更偏向短中周期，适合作为局部触发和突破确认主线。
- `iterA1`（若内部口径称为 `iter9a1`，则对应这轮 A 股适配训练结果）进一步支持了上述判断，并给出了较清晰的 A 股验证数据。

因此，当前阶段的主要研究价值不在于单项指标的继续提升，而在于已逐步确认两类信号各自的周期定位与功能分工。这一结论将直接支撑后续论文写作、平台路线评估与应用表达层设计。
