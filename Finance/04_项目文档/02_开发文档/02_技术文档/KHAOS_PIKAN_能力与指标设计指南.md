# KHAOS（PI-KAN）能力与指标设计指南

## 1. 训练后模型能提供什么信息

### 1.1 可观测输出
- **Force（驱动力）**：基于 `hurst + entropy + volatility` 的蒸馏函数输出，表示短期回归/反转驱动方向与强度。
- **Energy（EKF Z-Score）**：当前价格相对状态估计的偏离能量，表示“冲击是否足够大”。
- **Regime Validity（状态可交易性）**：由波动门（`vol_gate`）与结构门（`hurst_gate`）共同判断当前是否允许执行反转。
- **Main/Bias（势能偏置）**：价格相对 EKF 轨道的位置，表示当前环境偏空/偏多。
- **Composite（复合评分）**：融合 Force 与 Energy 的总评分，用于快速判读。
- **RegimeScore（状态质量）**：衡量当前结构环境质量，帮助判断信号可信度。

### 1.2 业务含义映射
- **能回答的问题**
  - 当前是否属于可执行反转的状态？
  - 当前偏向多还是空？
  - 触发是否来自“真实冲击”还是噪声？
  - 当前信号强度是否足够支持执行？
- **不能直接回答的问题**
  - 长周期趋势预测的终点价位。
  - 基本面变化造成的中长期定价重估。

### 1.3 能力边界
- 该模型是**相变/反转筛选器**，不是传统趋势跟随器。
- 在低波动或强单边区间，应该更常见 `BLOCKED`，这是正常行为。
- 信号表达的是条件优势，不是确定性预测。

## 2. 如何针对模型能力设计指标

## 2.1 设计总原则
- **先状态，后方向，再触发**：先判断可交易性，再判断偏向，再看入场脉冲。
- **一眼三问**：是否可做？做多还是做空？为什么是现在？
- **证据可追溯**：每个信号都能追溯到 Force、Energy、Regime 三类证据。

### 2.2 分层架构（推荐）
- **执行层（主图）**
  - BUY/SELL 脉冲标签
  - Top/Bottom Zone 持续区间
  - Regime 背景（可选）
- **解释层（副图）**
  - Energy 柱状图（冲击强度）
  - Main 线（偏置方向）
  - Composite 线（综合强度）
- **摘要层（状态表）**
  - State / Main / Force / Energy / Composite / RegimeScore

### 2.3 视觉语义（固定规范）
- **绿色**：可执行、偏多、正向确认
- **红色**：屏蔽、偏空、负向确认
- **蓝色系**：多头强度渐进
- **橙色系**：空头强度渐进
- **灰色背景**：高混沌或观望区

### 2.4 参数分组（防止误调）
- **Physics Core**：`alpha`、`hurst_fast`、`hurst_slow`
- **Regime Analysis**：`pe_len`、相关结构窗长
- **Signal Logic**：`sigma_trig`、`sigma_exit`、`vol_gate`、`hurst_gate`
- **Display**：显示开关与平滑参数（不改变核心决策逻辑）

## 3. 推荐指标形式（你可直接采用）

### 3.1 面板布局
- **主图最少元素**：BUY/SELL + Zone + 轻量背景
- **副图核心三元组**：Energy（柱）+ Main（线）+ Composite（线）
- **右上角状态卡**：只放关键数值，禁止堆叠冗余文本

### 3.2 触发解释文案（示例）
- **BUY**：`Regime VALID + Force>阈值 + Energy<-阈值`
- **SELL**：`Regime VALID + Force<-阈值 + Energy>阈值`
- **BLOCKED**：`Regime 不通过（波动或结构门未满足）`

### 3.3 可解释性增强建议
- 对每个信号点提供“触发原因短码”（如 `R+F+E`）。
- 在状态表中显示当前门控开关状态（`VolGate / HurstGate`）。
- 保留 Display 开关便于不同交易风格用户做视觉简化。

## 4. 手动验证时应重点看什么
- **一致性**：同类行情下，State 与信号行为是否稳定一致。
- **筛选性**：在单边下跌/上涨中，BLOCKED 是否显著增多。
- **时效性**：Energy 峰值附近是否更容易出现有效反转。
- **可操作性**：信号密度是否可执行，是否出现过多噪声触发。
