## 当前量化观察

### 1. `iterA5` 到目前为止没有显示出可替换 `iterA3` 的迹象

- `iterA3 overall_model.composite = 0.4187`
- `iterA4 overall_model.composite = 0.3906`
- `iterA5` 当前 best 的合理推断值约为 `0.3766`
  - 依据：
    - `epoch 14 composite = 0.3766`
    - `epoch 15` 出现 `未超过最优，连续未改进轮数: 1/8`
    - 这说明当前 best 大概率在 `epoch 14` 已刷新
- 也就是说：
  - 相比 `iterA3` 仍落后约 `0.0421`
  - 相比 `iterA4` 也仍落后约 `0.0140`

### 2. `breakout / reversion` 主任务几乎没有被 `5m + iterA5 架构` 拉起来

- `iterA3`
  - `breakout_f1 = 0.4632`
  - `reversion_f1 = 0.4821`
- `iterA4`
  - `breakout_f1 = 0.4636`
  - `reversion_f1 = 0.4778`
- `iterA5` 当前 best 近似可读作：
  - `breakout_f1 ≈ 0.4622`
  - `reversion_f1 ≈ 0.4744`

结论：

- `breakout` 没有因为新增 `5m` 监督锚点而抬升。
- `reversion` 仍低于 `iterA3`，也没有赢过 `iterA4`。
- 这说明本轮新增的数据密度和结构复杂度，至少到 `epoch 16` 为止，还没有转成公共主任务收益。

### 3. 方向分支明显变弱，但公共输出没有获得等价补偿

- `iterA4 direction_macro_f1 = 0.9006`
- `iterA5` 当前 `epoch 14~16` 的 `direction_macro_f1` 约在 `0.7787 ~ 0.7870`

结论：

- `iterA5_multiscale` 中新增的独立 `public_reversion_mixer/head`，很可能削弱了蓝/紫方向分支的可分性。
- 但公共 `reversion` 并没有因此明显变强。
- 这更像是“把方向信息打散了”，而不是“把方向信息更高效地转成了公共反转分数”。

## 训练曲线特征

### 1. 模型很早就进入平台期

- `epoch 1 composite = 0.3525`
- `epoch 6 composite = 0.3682`
- `epoch 10 composite = 0.3736`
- `epoch 14 composite = 0.3766`
- `epoch 15 composite = 0.3757`
- `epoch 16 composite = 0.3759`

结论：

- 前 `10` 个 epoch 基本完成了大部分可见提升。
- 后续只是在 `0.375 ~ 0.377` 一带窄幅波动。
- 这更像是“结构/目标函数已经吃满当前信息”，而不是“训练轮数不够”。

### 2. `val_loss` 的波动没有转化为更好的 `composite`

- `epoch 14 val_loss = 16.9410`
- `epoch 15 val_loss = 28.2645`
- `epoch 16 val_loss = 16.9640`
- 但对应 `composite` 仍基本卡在 `0.375x`

推断：

- 当前 `loss_profile='iterA5'` 中更重的 `event_gap / hard_negative / continuation_suppression`，可能放大了部分文件上的损失波动。
- 但这种波动没有换来更高的事件识别质量。
- 这意味着本轮 loss 权重很可能已经出现“优化目标和最终选择目标不完全同向”的现象。

## 当前最值得怀疑的问题

### 1. `5m` 扩容带来的主要是样本量和噪声，不是有效监督增益

证据：

- `iterA5` 已经引入 `5m`，但整体 composite 没有超过 `iterA4`。
- 当前滚动日志里，很多 `5m` 文件的即时 `Composite` 仍常见在 `0.37 ~ 0.39` 一带，并没有持续压过 `1d / 240m` 的高质量片段。
- 开收盘四类 `5m` bar 已额外乘上 `0.75` 边界降权，但假信号率仍没有看到决定性改善。

推断：

- `5m` 在 A 股里依旧混入了较多会话边界、撮合噪声和局部脉冲。
- 现有边界降权不足以把这些噪声真正隔离掉。
- 本轮更像是“把高频样本喂进来了”，但模型没能学到比 `15m/60m` 更可靠的公共相变纹理。

### 2. `60m` 核心短板大概率没有被修复

已知事实：

- `iterA4` 最弱周期就是 `60m`，其 composite 只有 `0.3492`。
- `iterA5` 本轮虽然提高了 `60m` 权重、保留 `240m`、新增 `5m`，但截至 `epoch 16` 整体 composite 反而还低于 `iterA4`。

推断：

- 如果 `60m` 真被显著修好，整体模型分数不太可能还停留在 `0.376x`。
- 因此当前最合理的判断仍是：
  - `60m` 问题没有解决；
  - 只是被新增 `5m` 和更复杂的 reversion 结构“盖住了表面症状”。

### 3. 公共 `reversion` 独立分支设计方向未必错，但耦合方式可能错了

证据：

- 本轮增加了 `public_reversion_mixer` 与 `public_reversion_head`。
- 结果却是：
  - 公共 `reversion_f1` 没有起来；
  - 方向分支 `macro_f1` 明显下降。

推断：

- 问题不一定在“是否要给公共 reversion 独立分支”。
- 更可能在：
  - 当前 gate 融合比例不合适；
  - 公共 reversion 分支和蓝/紫方向分支之间的约束变弱了；
  - 独立分支开始学习一类和 teacher/THS 不完全一致的捷径。

### 4. 当前 dual-threshold 设计更像“验收门槛”，还没有成为“训练内纠偏机制”

事实：

- 本轮的双门槛只用于训练完成后是否升级默认版：
  - `overall_model.composite >= 0.4187`
  - `calibrated_ths.test_objective >= 0.4448`
  - `60m composite >= 0.4032`
- 但训练中的 checkpoint 选择仍主要看模型侧 score profile。

推断：

- 当模型已经早早陷入 `0.376x` 平台期时，训练过程里没有任何机制提示“这一轮虽然还能降 loss，但已经不可能过升级门槛”。
- 这会导致：
  - 训练继续跑；
  - 但优化方向不再服务于真正的版本升级目标。

## 当前判断

- 截至 `2026-04-07` 本次中期复盘，`iterA5_ashare` 不应被视为接近升级成功。
- 当前更像是一次“把 `5m` 真正接进主训练后，对架构和 loss 假设做了验证”，而不是一次成功提分。
- 本轮暴露出的事实是：
  - `5m` 不是天然正增益；
  - 独立公共 reversion 分支并没有自动修复公共输出；
  - `60m` 仍极可能是主阻塞点；
  - 当前 loss/score 组合很可能已经不再匹配本轮最需要修复的问题。

## 建议的下一步排查顺序

### 1. 不要直接继续堆 epoch 期待自然变好

- 目前最像“结构性平台”，不是“训练轮次不足”。
- formal 即使继续跑完，也大概率只是在确认这件事，而不是逆转结论。

### 2. 先做最小 ablation，把问题拆开

- ablation A：
  - 保留 `iterA5` 数据集与 `5m`
  - 架构退回 `iterA4_multiscale`
  - loss 退回 `iterA4`
  - 验证“问题到底来自 `5m` 还是来自新 reversion 结构/新 loss”
- ablation B：
  - 保留 `iterA5_multiscale`
  - 但把 `5m` 从主 checkpoint 训练中拿掉，或进一步大幅降权
  - 验证“结构是否本身可用，只是被 `5m` 噪声拖坏”
- ablation C：
  - 在当前 best checkpoint 上先补做分周期 OOS 分析
  - 重点确认 `60m` 是否仍显著低于 `15m / 240m / 1d`

### 3. 若继续保留 `5m`，应该重新定义它在训练中的角色

- 倾向把 `5m` 当成：
  - 辅助监督
  - warmup/预训练锚点
  - 或仅参与局部事件纹理学习
- 而不是直接与 `15m / 60m / 240m / 1d` 等权进入同一主 checkpoint 竞争。

### 4. 下轮必须把“问题定位输出”前置，而不是只看最终 composite

- 至少要新增：
  - 分周期 composite 曲线
  - `5m / 15m / 60m / 240m / 1d` 的假信号率对比
  - 公共 reversion 分支与蓝/紫分支的 gate 使用分布
- 否则下一轮仍然会知道“结果差”，但不知道是数据、结构还是 loss 在拖后腿。
