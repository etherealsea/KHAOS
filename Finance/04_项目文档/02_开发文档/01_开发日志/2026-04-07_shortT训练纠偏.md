# 2026-04-07 shortT 训练纠偏

## 背景

- 在 `iterA5` 阶段复盘后，已完成一轮“短周期增强”方向的最小实现与 smoke 验证。
- 该实现最初命名为 `shortT_breakout_v1`，其核心目的是验证：在保留 `iterA4_multiscale` 稳定主干的前提下，强化 `5m/15m/60m` 是否能提升短周期可用性。
- 用户最新明确要求是：**不能把本轮目标理解为“只优先拉爆发核”**，而应改为：
  - `breakout` 与 `reversion` 同步推进；
  - 重点提升模型本身的准确度、F1、假信号控制和分周期稳定性；
  - 同时保证结果继续可映射到同花顺终端，不把 teacher 做成无法落地的研究型输出。

  - `directional_floor_mean`
  - `blue_over_purple_violation`
  - `purple_over_blue_violation`
  - `public_below_directional_violation`
  - `continuation_public_violation`

结论：

- 现在训练失败时，不再只是看到总 `composite` 变差，而是能直接定位是：
  - 某个周期掉队；
  - 某个分支失衡；
  - 某类物理约束被频繁违反；
  - 或某个 gate 把方向信息冲淡了。

### 2. 短周期样本权重增强

- 已在 `ashare_dataset.py` 增加 `dataset_profile='shortT_v1'`。
- 当前实现对训练样本做了三类增强：
  - 提高 `5m/15m/60m` 的时间框架权重；
  - 降低 `240m/1d` 的主导程度，但不移除；
  - 对 `5m` 开收盘边界样本施加更强惩罚，减少会话边界噪声影响。
- 同时加入了面向事件质量的样本加权：
  - `breakout_event`
  - `breakout_aux`
  - `breakout_hard_negative`

结论：

- 这一步的本意是让模型更重视短周期局部纹理，尤其是更适合 A 股“做 T”场景中的快速节奏切换。
- 但该版本对 `breakout` 倾斜偏强，不能直接代表本轮最终训练方向。

### 3. 约束型 teacher 可行域已接入

- 已在 `loss.py` 中加入 `constraint_profile='teacher_feasible_v1'`。
- 当前显式约束包括：
  - `blue_context -> blue >= purple + 0.12`
  - `purple_context -> purple >= blue + 0.12`
  - `reversion_event -> reversion >= directional_floor + 0.10`
  - `continuation_pressure -> reversion <= directional_floor + 0.05`

结论：

- 这一步对应“可行域/线性约束”思路在 KHAOS 中的落地版本。
- 它的作用不是让模型去追求某一头分数更高，而是把输出限制在更一致、更可解释、也更接近终端规则逻辑的状态空间里。
- 这与 KHAOS 一直强调的“物理约束优先于纯统计拟合”是同方向的。

## 本次纠偏后的正式结论

### 1. `shortT_breakout_v1` 只能视为探索分支，不能视为当前主线版本

- 它验证了“短周期增强 + 约束日志 + 新选模逻辑”这条工程链路是通的。
- 但它的目标函数明显偏向 `breakout`，与用户当前要求的“爆发核与回归核同步推进”不完全一致。

### 2. 下一轮训练的主目标应从“短周期爆发优先”改为“短周期增强下的双核同步提分”

下一轮正确目标应是：

- 不牺牲 `reversion_f1` 去换 `breakout_f1`；
- 不牺牲 `direction_macro_f1` 去换局部短周期收益；
- 不牺牲 `240m/1d` 的背景稳定性去换 `5m` 的表面敏感度；
- 不牺牲同花顺可映射性去换纯研究环境里的 teacher 分数。

### 3. 同花顺兼容性必须继续作为硬约束

- 不新增无法映射到 THS 的原始输入字段；
- 继续保留对外 `breakout / reversion` 双输出接口；
- 方向分支、约束、gate 可以在 teacher 内部存在，但必须服务于终端可还原的信号逻辑；
- checkpoint 判断不能只看 teacher 侧总分，后续仍要保留 THS 对齐校验。

## 对下一轮重启训练的建议

### 1. 保留本轮已经完成的“基础设施升级”

- 保留分周期日志；
- 保留 gate/constraint 统计；
- 保留 `teacher_feasible_v1` 约束框架；
- 保留短周期增强的数据入口。

### 2. 重启前应纠偏的方向

- 不再把 `shortT_breakout_v1` 当作默认正式候选；
- 将短周期目标从“爆发核偏置”改成“breakout/reversion 同步提分”；
- 让 `5m` 的作用更偏向：
  - 局部纹理学习；
  - 假突破/假反转过滤；
  - 做 T 场景的短期辅助判断；
  - 而不是主导整轮 checkpoint 选择。

### 3. 下一轮最该盯的指标
