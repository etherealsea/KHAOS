## iterA2 已完成但仍不够好的原因

### 1. iterA2 实际做了哪些优化

- `iterA2` 对 `PI-KAN` 网络主体本身改动很小，核心收益主要来自训练体系升级，而不是新 backbone。
- 数据侧优化：
  - 引入 A 股可交易性约束，持续压制一字板、低流动性、长停牌缺口、公告跳空、隔夜大跳空、午休制度性断档的误导作用。
  - 在样本中加入 `breakout/reversion` 的 event mask 与 hard-negative mask。
  - 新增方向化 reversion 上下文标记，用于区分蓝色分支和紫色分支候选。
- 损失与选择口径优化：
  - 延续 `p3/p4/p6/p7 + event_gap + hard_negative` 的 `PhysicsLoss` 框架。
  - best checkpoint 选择从偏 `corr` 改为偏事件质量，重点看 `F1`、硬负样本误报率和信号频次偏差。
- THS 对齐优化：
  - 新增 Python 版 `KHAOS_THS_CORE` 代理计算器。
  - 完成 `BK_EVT_TH / RV_EVT_TH / RES_NODE / ENT_BULL / ENT_BEAR / MLE_GATE / DIR_GAP` 及分支权重的微调。

### 2. iterA2 实际训练了多少 epoch

- smoke run：`1` 个 epoch。
- formal run：`24` 个 epoch。
- 训练方式：纯 A 股、纯 scratch 启动，没有 warm-start。
- 早停情况：**没有触发 early stop**，实际完整跑满 `24/24` 个 epoch。
- 最佳轮次：`epoch 24`。
- 关键记录：
  - `best_score = 0.3635609964`
  - `best_val_loss = 16.0406351688`

### 3. iterA2 结果与含义

- `model composite = 0.3679`
  - 含义：模型综合事件质量分，主要看 `breakout/reversion` 的 `F1`、硬负样本误报率和少量相关性。
  - 判断：比 `iterA1` 的 `0.3609` 有提升，但提升不够大。
- `breakout_f1 = 0.4633`
  - 含义：真正爆发样本的识别能力。
  - 判断：相比 `iterA1` 的 `0.4621` 几乎没有本质改善。
- `reversion_f1 = 0.4864`
  - 含义：真正反转样本的识别能力。
  - 判断：比 `iterA1` 的 `0.4767` 更好，但仍未站上 `0.50`。
- `THS test objective = 0.4380`
  - 含义：同花顺代理回放下的整体可用性综合分。
  - 判断：比 `iterA1` 的 `0.3985` 提升明显，说明公式对齐方向有效。
- `THS label macro F1 = 0.3813`
  - 含义：`breakout / blue / purple / neutral` 四状态对标签的平均识别质量。
  - 判断：仍偏低，终端状态识别还不够稳。
- `THS teacher macro F1 = 0.5231`
  - 含义：同花顺公式对 teacher 状态的跟随度。
  - 判断：已改善，但还没有达到足够稳妥的落地水平。

### 4. 为什么 iterA2 还不够好

- 爆发识别能力几乎停滞，`breakout_f1` 没真正拉起来。
- 反转分支虽然提升，但 continuation 和假反转仍较多。
- THS 可用性进步主要来自参数校准，网络本体对“蓝 / 紫方向区分”的建模仍偏弱。
- 因此下一轮不能只靠继续多跑 epoch，必须直接增强 `PI-KAN / KHAOS_KAN` 结构本身。

## iterA3 本轮执行内容

### 1. 对 PI-KAN / KHAOS_KAN 做了哪些优化

- 保留不变：
  - 仍使用当前 `14` 维 physics 特征。
  - 主输出接口仍是双头 `breakout / reversion`。
  - 仍保持与同花顺单主线语义一致，不把 teacher 退化成公式分类器。
- 结构升级：
  - 增加多尺度时序汇聚，分别建模全窗、近 `10` 根和近 `5` 根上下文。
  - 新增 `shared / transition / reversion` 三类状态混合器，避免一个共享状态同时承担全部任务。
  - 在 reversion 内部拆出 `blue_reversion_head` 与 `purple_reversion_head` 两个方向子头。
  - 新增 `direction_gate`，将蓝色分支与紫色分支融合成对外唯一的 `reversion` 分数。
- 数据与标记升级：
  - 保留 `iterA2` A 股可交易性约束。
  - 新增 `continuation_pressure` 标记，用来识别“更像趋势延续而不是反转”的样本。
  - `event_flags` 从 `6` 维扩展到 `7` 维，并改成具名索引读取。
- 损失升级：
  - 保留 `p3/p4/p6/p7 + event_gap + hard_negative` 主体。
  - 新增 `direction_consistency_loss`，约束蓝色上下文样本由蓝色分支占优，紫色上下文样本由紫色分支占优。
  - 新增 `continuation_suppression_loss`，额外压制 continuation 场景下的假反转。
- 评估升级：
  - checkpoint 综合分加入 `direction_macro_f1`，不再只看 breakout/reversion 两类总分。
  - 新增 `iterA3_ashare` 版本化 runner、analyzer 和 THS 校验入口。

### 2. 训练过程中实际发生了什么

- smoke run 首次执行时暴露了一个真实问题：
  - 验证阶段方向分支输出使用了 `bfloat16`，直接转 `numpy` 报错。
  - 已修复为先转 `float32` 再落盘，之后 smoke 重新通过。
- smoke run：
  - `1` 个 epoch。
  - 结果：训练链路、方向分支、断点文件、best/final/resume 均成功生成。
- formal run：
  - `32` 个 epoch。
  - 训练方式：从 `iterA3_ashare` 全新目录 scratch 启动，没有复用 `iterA2` 权重。
  - 早停情况：**没有 early stop**，实际完整跑满 `32/32` 个 epoch。
  - 最佳轮次：`epoch 32`。
  - 关键记录：
    - `best_score = 0.4171261505`
    - `best_val_loss = 16.4522478658`
    - `processed_files = 72`

## iterA3 当前结果与直观判断

### 1. 模型侧结果

- `iterA3 model composite = 0.4187`
  - 对比：`iterA2 = 0.3679`
  - 含义：综合事件质量、硬负样本控制、方向识别和少量相关性的总分。
  - 判断：**显著提升**，说明这轮结构升级是有效的。
- `iterA3 breakout_f1 = 0.4632`
  - 对比：`iterA2 = 0.4633`
  - 含义：真正爆发样本的识别能力。
  - 判断：**基本持平**，爆发头仍未实现实质突破。
- `iterA3 reversion_f1 = 0.4821`
  - 对比：`iterA2 = 0.4864`
  - 含义：真正反转样本的识别能力。
  - 判断：**略有回落**，说明这轮结构把一部分收益转移到了方向区分和整体稳定性上，但纯反转命中率还需继续优化。
- `iterA3 direction_macro_f1 = 0.9469`
  - 含义：蓝色分支与紫色分支在方向上下文样本上的平均区分能力。
  - 判断：**这是本轮最大新增能力**，说明网络已经学会比较稳定地区分蓝/紫方向。
- `breakout_corr = 0.2691`
  - `reversion_corr = 0.3610`
  - 含义：模型输出与连续 target 强度的线性相关性。
  - 判断：相关性进一步提升，但现在不是首要目标，只作为辅助参考。

### 2. THS 对齐结果

- `iterA3 default THS objective = 0.4303`
  - 含义：不调参时的同花顺代理综合可用性。
  - 判断：低于 `iterA2` 已校准版本，说明结构升级后仍需要同步做公式参数校准。
- `iterA3 calibrated THS objective = 0.4448`
  - 对比：`iterA2 = 0.4380`
  - 含义：校准后同花顺代理的整体可用性综合分。
  - 判断：**小幅超过 iterA2**，已形成可推广候选。
- `iterA3 calibrated THS label macro F1 = 0.3943`
  - 对比：`iterA2 = 0.3813`
  - 含义：四状态对标签的平均识别质量。
  - 判断：**继续提升**，终端读图层更接近真实标签。
- `iterA3 calibrated THS teacher macro F1 = 0.5206`
  - 对比：`iterA2 = 0.5231`
  - 含义：同花顺公式对 teacher 状态的跟随度。
  - 判断：**略低于 iterA2**，说明公式对新结构 teacher 的跟随还没完全同步到位。

### 3. 这轮是不是赢了 iterA2

- 模型综合分：赢了。
  - `0.3679 -> 0.4187`
- THS 综合目标分：赢了。
  - `0.4380 -> 0.4448`
- 纯 breakout F1：没有赢，只是持平。
- 纯 reversion F1：没有赢，略有下降。
- THS label macro F1：赢了。
- THS teacher macro F1：没有赢，略有下降。

### 4. 当前结论

- `iterA3` 不是失败版本，**整体是有效升级**。
- 这轮最明确的新增价值不在“单纯提高反转 F1”，而在：
  - 明显拉高总 composite；
  - 让蓝 / 紫方向分支变得非常稳定；
  - 让 THS 标签态质量继续改善。
- 但它还不是最终满意版本，因为：
  - breakout 头没有被真正拉起来；
  - reversion F1 还没有达到预期；
  - THS teacher 跟随度没有同步超过 `iterA2`。
