# 开发工作日志 (Development Log)

## 2026-03-09 PI-KAN v4.0 高波动模式与指标可解释性重构 (High-Vol Mode & Interpretable Refactoring) - [Completed]

### 1. 战略调整 (Strategic Shift)
- **背景**: 用户指出 v3.0 模型在 5m 短周期上反应滞后，且在处理外汇等均值回归型资产时表现平庸。同时，KHAOS 指标的可视化被批评为“黑箱”，缺乏直观的物理状态反馈。
- **决策**:
    - **资产聚焦**: 彻底剔除外汇数据，**PI-KAN v4.0** 仅在 BTC, ETH, SOL, 黄金, 原油, 标普/纳指期货等**高波动资产**上训练。
    - **激进调优**: 降低熵阻尼 (Entropy Damping)，大幅提高动量 (Momentum) 权重，打造专为捕捉 5m 级别剧烈反转设计的“激进版”模型。
    - **白箱化**: 彻底重构指标 UI，引入“数据窗口”支持和“动态置信度”视觉，让用户看懂背后的物理逻辑。

### 2. 核心技术升级 (Technical Upgrades)
- **PI-KAN v4.0 模型重训**:
    - **训练集**: 纯化为 High-Vol Crypto/Commodities/Futures。
    - **Loss 调整**: `p3_entropy` 权重从 0.05 降至 0.01 (减少滞后)，`p1_momentum` 从 0.1 升至 0.2 (增强趋势跟随)。
    - **结果**: 5m 周期上的响应速度显著提升，消除了“趋势已出，指标未动”的尴尬。
- **KHAOS 指标重构**:
    - **可视化**: 移除了被诟病的垂直色带，改为 **动态梯度填充 (Dynamic Gradient Fill)**，用颜色深浅直接映射信号置信度。
    - **交互性**: 实现了 **Cursor-Tracking Labels**，在图表上悬浮显示当前 K 线的物理参数，替代难找的侧边栏数据窗口。
    - **可解释性**: 明确定义了“灰色主线 = 物理平衡态 (EKF 预测准确)”的概念，并在文档中详细阐述。

### 3. 项目治理 (Project Governance)
- **归档**: 创建 `Finance/archive/v3_legacy/`，移除了所有过期的 v3.0 模型和训练日志。
- **文档**: 更新了 `KHAOS_USER_GUIDE.md`，新增了“高波动模式特别说明”和“信号分层判断指南”。
