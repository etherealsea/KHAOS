# 2026-04-04 iterA4 追加记录

## 本轮目标

- 在 A 股正式训练中新增 `240m(4h)` 周期。
- 让训练数据生成逻辑更贴合同花顺的 A 股 bar 语义。
- 基于 `iterA3_ashare` 继续迭代出 `iterA4_ashare`，但不覆盖 `iterA3`。
- 自动完成 smoke、formal train、结果分析与日志归档。

## 本轮对 PI-KAN / KHAOS_KAN 做了哪些升级

- 新增正式训练周期：`15m / 60m / 240m(4h) / 1d`。
- 修正 A 股 `240m` 数据生成逻辑：
  - 不再用按时钟 `4h` 的通用重采样。
  - 改为按 A 股单个交易日 `240` 交易分钟聚合，时间戳落在当日尾 bar，贴合同花顺 `4h/240m` 语义。
- 修正训练数据源优先级：
  - 目标周期优先使用同周期原始数据。
  - 若没有同周期数据，再使用最接近的更低周期数据重采样。
  - 这避免了 `60m`/`1d` 长期被 `15m` 粗暴重采样覆盖。
- 新增 `iterA4_multiscale` 结构：
  - 保留双头公开接口 `breakout / reversion` 不变。
  - 在 `iterA3` 蓝/紫方向专家基础上，新增 `public_reversion_gate`，让公共 `reversion` 分数不再只靠单一方向门控混合，而是能吸收 `max(blue, purple)` 的强证据。
  - 新增 `breakout_residual_gate`，把短中期突变上下文补给 breakout 分支，尝试提升 breakout 对快速相变的敏感度。
- 新增 `iterA4` 训练约束：
  - `loss_profile = iterA4`，提高 `breakout_event_gap`、`breakout_hard_negative`、`continuation_suppression` 权重。
  - 下降 `direction_consistency` 权重，避免模型只顾把蓝/紫分支做得很准，却牺牲公共 `reversion` 输出。
- 新增 `score_profile = iterA4_event_focus`：
  - best checkpoint 选择进一步偏向 `breakout / reversion` 事件质量，降低方向分支对最佳模型选择的主导性。
- 新增版本化入口与产物：
  - `run_iterA4_ashare_train.py`
  - `analyze_iterA4_ashare_results.py`
  - `validate_iterA4_ths_proxy.py`
  - 新权重、日志、分析报告全部写入 `iterA4_ashare` 独立目录。

## iterA4 最终结果与含义

- `model composite = 0.3906`
  - 含义：模型侧综合分，主要衡量 `breakout/reversion` 事件质量、硬负样本误报控制、少量相关性，以及少量方向能力。
  - 对比：低于 `iterA3` 的 `0.4187`
  - 结论：本轮没有在 teacher 主任务上赢过 `iterA3`。
- `breakout_f1 = 0.4636`
  - 含义：真实 breakout 样本的识别质量，越高表示真正爆发点更容易被抓到。
  - 对比：接近 `iterA3` 的 `0.4632`
  - 结论：几乎持平，没有形成有效突破。
- `reversion_f1 = 0.4778`
  - 含义：真实 reversion 样本的识别质量，越高表示真正反转更容易被抓到。
  - 对比：低于 `iterA3` 的 `0.4821`
  - 结论：公共 reversion 输出仍未达到预期。
- `direction_macro_f1 = 0.9006`
  - 含义：蓝色分支和紫色分支在方向上下文中的平均区分能力。
  - 对比：低于 `iterA3` 的 `0.9469`
  - 结论：本轮为了让公共信号更实用，牺牲了一部分方向分支纯判别力，但主信号收益也没有补回来。
- `THS default test objective = 0.4303`
  - 含义：不改同花顺默认参数时，Python 代理回放下的整体可用性分数。
  - 对比：与 `iterA3 default` 基本持平。
  - 结论：默认参数下并没有明显更好。
- `THS calibrated test objective = 0.4441`
  - 含义：对同花顺参数做微调后，公式代理与 teacher/标签状态的综合贴合度。
  - 对比：略低于 `iterA3 calibrated` 的 `0.4448`
  - 结论：参数校准后仍有可用性，但没有形成新的最优版本。
- `THS calibrated label_macro_f1 = 0.3930`
  - 含义：`breakout / blue / purple / neutral` 四状态对标签的平均识别质量。
  - 对比：略低于 `iterA3` 的 `0.3943`
  - 结论：终端读图层面没有进一步抬升。
- `THS calibrated teacher_macro_f1 = 0.5209`
  - 含义：同花顺公式对 teacher 状态的跟随度。
  - 对比：略高于 `iterA3` 的 `0.5206`
  - 结论：公式对 teacher 的跟随边界略有改善，但不足以抵消主模型退步。

## 分周期观察

- `240m(4h)`：
  - `breakout_f1 = 0.4987`
  - `reversion_f1 = 0.5146`
  - 说明：新增 `4h` 周期本身是有效的，质量接近 `1d`，说明 A 股 `240m` 按交易日聚合的逻辑是成立的。
- `1d`：
  - `breakout_f1 = 0.4984`
  - `reversion_f1 = 0.5119`
  - 说明：长周期仍然最稳定。
- `15m`：
  - `breakout_f1 = 0.4660`
  - `reversion_f1 = 0.4842`
  - 说明：短周期基本稳定，但没有把新增结构收益转成主信号提升。
- `60m`：
  - `breakout_f1 = 0.4223`
  - `reversion_f1 = 0.4311`
  - 说明：`60m` 仍然是最薄弱、且本轮退步最明显的周期，是当前正式版无法升级的核心阻塞点。

## 本轮结论

- `iterA4` 不是废训练。
- 它证明了两件关键事情：
  - A 股 `4h/240m` 周期可以稳定接入，且数据生成逻辑必须按交易日而不是按时钟 `4h` 处理。
  - 只靠“加强公共 reversion 聚合 + 调损失权重”还不足以把主模型性能整体推过 `iterA3`。
- 但 `iterA4` 也明确没有达到替换条件：
  - teacher 主任务综合分下降；
  - calibrated THS 总体也没有稳定赢过 `iterA3`；
  - `60m` 周期拖累明显。
- 因此当前默认版本仍应保留 `iterA3_ashare`，`iterA4_ashare` 作为 `4h` 接入与 bar 对齐修正后的实验基线保留。

## 下一轮 iterA5 建议方向

- 重点一：单独救 `60m`
  - 为 `60m` 建立单独的 checkpoint 观察口径。
  - 提高 `60m` 在训练抽样中的存在感，但不要让它把 `15m/1d/240m` 一起拖坏。
- 重点二：继续分离“公共 reversion 分数”和“蓝/紫方向专家”
  - 这次的 `public_reversion_gate` 方向是对的，但还不够强。
  - 下一轮更适合加“公共 reversion 专用上下文”而不是继续单纯调蓝/紫混合方式。
- 重点三：把 `240m` 当作 `1d` 与 `60m` 之间的桥
  - 让 `240m` 参与中周期稳定化，而不是只作为新增周期被动混入。
- 重点四：checkpoint 选择改成“双门槛”
  - 模型 composite 不低于 `iterA3`
  - calibrated THS objective 也不低于 `iterA3`
  - 两者同时满足才允许升级默认参数。
