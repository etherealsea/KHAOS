# 开发日志 (2026-03-20)：PI-KAN V3 架构革新与实盘信号失效的深度反思

## 一、 今日工作概述
今日的核心工作聚焦于将 PI-KAN（物理信息增强神经网络）升级至 V3 版本，并尝试将模型训练提取的物理规律转化为 TradingView (Pine Script) 实盘指标。整个过程经历了从架构重构、训练目标调整，到规则提取与 UI/UX 迭代的完整闭环。然而，在最终的实盘映射阶段，发现生成的指标信号准确率极低，引发了对“模型理论到实盘工程”转化过程的深刻反思。

## 二、 核心观点与技术革新

### 1. RA-PI-KAN 架构的提出
在模型设计上，我们成功融合了注意力机制与物理约束，提出了 RA-PI-KAN (Residual Attention PI-KAN) 架构。关于该架构的详细输入输出设计、特征工程说明及损失函数定义，请参阅专门的架构设计文档：**[开发文档_PI-KAN_V3_架构设计.md](./开发文档_PI-KAN_V3_架构设计.md)**。
核心革新点包括：
*   **物理特征降维**：摒弃了冗余的 SMC/PA 信号，将特征精简至 8 维核心物理状态（如 EKF 速度/残差、Hurst 指数、MLE、熵等）。
*   **网络拓扑更新**：通过 `AttentionResidualBlock` 增强了序列建模能力，并结合 KAN (Kolmogorov-Arnold Network) 的非线性样条拟合特性，大幅提升了对时间序列混沌相变的捕捉能力。

### 2. 训练目标的“有效性”重构
**这是一个重要的观念革新**：以往模型多以预测“绝对波动率”为目标，导致频繁在无方向的高波动震荡区产生误报。今日我们将目标重置为**预测“有效收益率” (Efficient Return = Forward Return * Efficiency Ratio)**。这一改动迫使神经网络去寻找不仅具有爆发力，更具备**趋势延续性**的关键拐点，过滤了大量无效的噪音波动。

### 3. 可视化理念的“极简像素风”进化
在 Pine Script 指标落地时，对 UI/UX 进行了颠覆性重构：
*   摒弃了传统的杂乱背景色、丝带和符号标记。
*   采用了**极简的 8-bit 像素风格**：使用透明极高的灰色阶梯线（stepline）作为底线，仅在判定发生物理相变时，直接在主线上叠加高亮的“粗方块”颜色块。
*   这种设计实现了“相变区间”的可视化保持（例如，处于趋势相变中时，红色方块会一直连贯），大幅提升了指标的盘面可读性。

## 三、 发现的关键问题：实盘信号准确度极低的根本原因

尽管提取出的规则（如：混沌爆发需满足 MLE > 0.27 且 Entropy > 0.70）在训练集表现优异，但在 TradingView 上的实盘映射却出现严重偏差（表现为完全变红或错失信号）。经过深度剖析，我们发现其根本原因在于**“物理代理指标的语义鸿沟”**与**“跨环境计算机制差异”**：

1. **严格物理计算 vs. 简易代理指标 (Proxy Mismatch)**
   *   **Python 端**：计算排列熵 (Permutation Entropy) 使用的是严格的微观状态空间概率分布计算；计算 MLE 可能涉及复杂的雅可比矩阵或相空间重构。
   *   **Pine Script 端**：受限于语言特性，Pine 中使用的“熵”是基于真实波幅 (TR) 与极差的比值代理 (`log10(sum_tr / rng)`)；“MLE”则是基于对数收益率散度的 EMA 平滑。
   *   **结论**：Python 中严格排列熵的 `0.70`，在数学绝对值上**完全等同于** Pine 中 TR/Range 代理的 `0.70`。直接将决策树提取的绝对阈值硬套在 Pine Script 的简易代理公式上，是导致信号彻底崩溃的元凶。

2. **全局归一化 vs. 流式窗口漂移 (Normalization Discrepancy)**
   *   Python 训练时，往往能够获取全局数据的统计特征，或者使用基于较长固定周期的滚动标准化。
   *   在 Pine 实盘流式处理中，如果未进行严格的分布对齐（例如动态 Z-score 缩放的周期与模型训练时的基准不同），会导致模型认为的“极值”在实盘图表上只是“常态”，反之亦然。

3. **多周期混合训练的阈值泛化灾难**
   *   训练数据集混合了 5m, 1h, 1d 等多周期数据。决策树提取出的统一阈值（如 MLE=0.27）可能是对多周期的妥协（平均值）。但在单一的 1 小时图表上，其真实 MLE 分布中位数可能在 0.15，这就导致硬阈值在特定周期下永远无法触发，或者永远处于触发状态（如用户反馈的“完全变成红色”）。

## 四、 下一步研究与修正计划

为解决上述“从理论到实盘脱节”的问题，后续开发不应再盲目修改 Pine 的硬参数，而应从系统工程角度入手：

1. **统一特征工程 (Feature Engineering Alignment)**
   *   废弃 Python 中过于复杂的物理计算包。**要求 Python 训练时的特征计算代码，必须 1:1 翻译自 Pine Script 中可实现的逻辑**。即：让神经网络直接学习 Pine 中 `ta.ema` 和 `sum_tr/rng` 代理指标的分布规律，而非学习学术界的理想物理模型。
2. **引入自适应阈值 (Adaptive Thresholds)**
   *   相变规则不应是绝对的 `0.27` 或 `0.70`，而应修改为**动态百分位位阈值**（例如：`当前 MLE > 过去 100 根 K 线 MLE 的 95th Percentile`）。这样可以彻底免疫时间周期切换和不同资产波动率基准带来的影响。
3. **消除未来数据隐患**
   *   全面排查 Python 端的 `calculate_lyapunov_proxy` 和目标 `Forward Return` 的对齐逻辑，确保决策树没有学习到任何包含“前视偏差 (Lookahead Bias)”的假性规律。

# PI-KAN V3.0 (RA-PI-KAN) 核心架构与训练目标设计文档

**更新日期**: 2026-03-20
**核心主题**: 融合注意力残差机制、李雅普诺夫混沌物理约束及“高效收益率”目标的神经网络重构

## 1. 架构概览 (Architecture Overview)

最新的 PI-KAN 架构已演进为 **RA-PI-KAN (Residual Attention Physics-Informed Kolmogorov-Arnold Network)**。
该架构摒弃了传统的多时间步展平 (Flattening) 输入，转而使用门控自注意力机制处理长程时序依赖，并将训练目标从单纯的“波动率预测”升级为“寻找具备高延续性的相变/反转点”。

数据流向如下：
`[多周期原始时序数据] -> [PhysicsLayer 物理特征提取] -> [AttentionResidualBlock 序列建模] -> [时序池化] -> [KAN 样条非线性拟合] -> [高效收益率预测]`

---

## 2. 数据源与多时间周期设计 (Data & Timeframes)

模型训练高度依赖于跨资产和多时间周期的泛化能力。为了确保模型能够学习到真实的物理相变特征，而非外汇市场的央行干预噪音，我们在数据层进行了严格的筛选。

### 2.1 训练时间周期 (Timeframes)
模型使用完整的跨周期数据以捕捉从高频微观结构到低频宏观趋势的演变，具体包括：
- **5m** (5分钟级别)
- **15m** (15分钟级别)
- **1h** (1小时级别)
- **4h** (4小时级别)
- **1d** (日线级别)

### 2.2 资产池构成 (Asset Pool)
我们严格**剔除了所有外汇（Forex）数据**（如 EURUSD, GBPUSD 等），并强制模型在以下核心资产上进行训练，以学习最纯粹的市场动力学特征：
1. **加密货币 (Crypto)**: BTC (比特币), ETH (以太坊)
2. **大宗商品 (Commodities)**: 黄金 (Gold/XAU/GC), 原油 (Crude Oil/WTI/CL)
3. **股指期货 (Index Futures)**: ES (标普500 E-mini), SPX (标普500现货/指数), NDX (纳斯达克100), DJI (道琼斯)

---

## 3. 输入特征精简设计 (Physics & Price Features)

为了提高模型的聚焦能力并减少噪声，去除了繁杂的 SMC/PA 信号（如 FVG、Sweep、PinBar），将单步输入特征严格精简为 **8 维**。

### 2.1 核心物理状态特征 (6 维)
这部分特征负责量化当前金融时间序列的“动力学相变状态”：
1. **Hurst 指数代理 ($H_t$)**: 衡量时间序列的长期记忆性（均值回归或趋势延续倾向）。
2. **当前波动率 (Volatility)**: 局部绝对收益率，表征当前能量水平。
3. **EKF 状态 ($V_{ekf}$)**: 扩展卡尔曼滤波提取的平滑趋势速度，代表系统的“惯性 (Inertia)”。
4. **EKF 残差 ($Res$)**: 实际价格与 EKF 预测的偏差，代表系统遭受的“突发冲击 (Shock)”。
   > *注：EKF 状态与残差必须同时存在，相变往往发生在巨大残差冲击打破原有系统惯性的时刻。*
5. **排列熵 (Permutation Entropy)**: 衡量系统局部的混乱度与复杂度。
6. **MLE 代理 (最大李雅普诺夫指数)**: 利用局部对数收益率散度计算，直接量化系统对初始条件的敏感性（混沌程度）。

### 2.2 核心价格特征 (2 维)
1. **Price Momentum (价格动量)**: 单步对数收益率，提供最无延迟的方向性基准。
2. **EMA20 Divergence (EMA20 偏离度)**: 价格偏离均线的程度，提供均值回归/极端超买超卖的结构信号。

---

## 3. 网络模块重构 (Neural Network Modules)

### 3.1 注意力残差序列建模 (Attention Residual Block)
引入了参考 Kimi 架构的带缩放因子的门控残差连接 (Gated Scaled Residuals) 与多头自注意力机制 (MHSA)。
- **作用**: 动态分配时间步的权重，捕捉长程依赖，同时通过 `output = x + alpha * gate(x) * f(x)` 极大地缓解深层网络中的梯度衰减和注意力坍塌。
- **输出**: 通过提取序列的最后一步 `x[:, -1, :]` 完成时序信息的上下文融合。

### 3.2 KAN 非线性拟合主干
在时序特征融合后，使用 Kolmogorov-Arnold Networks (KAN) 替代 MLP。
- **作用**: 利用边上的 B 样条函数进行非线性映射，在拟合金融物理系统中复杂的非线性流形（如分形、奇异吸引子）时，提供更高的参数效率和解释性。

---

## 4. 训练目标重构: 高效收益率 (Efficient Return)

为了满足“寻找带有延续性的反转或爆发点位”的业务诉求，训练目标从预测未来波动率更改为预测 **高效收益率 (Efficient Return)**。

**公式**: 
`Target = Forward_Return_H * Efficiency_Ratio`

- **Forward Return**: 未来 $H$ 步的对数收益率（决定方向与幅度）。
- **Efficiency Ratio (ER)**: 价格位移的直线距离与实际路径长度之比（$ER \in [0, 1]$）。
- **业务意义**: 只有当未来出现**幅度大且中途极少回调（高延续性的单边趋势）**时，Target 的绝对值才会很大。宽幅震荡或假突破会被 ER 惩罚趋近于 0。

---

## 5. 物理信息损失函数 (Physics-Informed Loss P1-P6)

主损失采用 `MSELoss`，在此基础上，由于预测目标变为“带有方向的延续性趋势”，物理惩罚项进行了同步逻辑反转：

**核心理念：在混沌/异常状态下，严禁模型预测“无趋势（死水一潭）”。**

- **P3 熵-相变一致性**: 
  当排列熵很高（系统处于无序状态，酝酿爆发）时，若模型预测的趋势绝对值极低，则施加惩罚。
- **P4 冲击-相变一致性**: 
  当 EKF 残差巨大（突发资金冲击）时，若模型预测未来无趋势，则施加惩罚。
- **P6 MLE 混沌边缘一致性**: 
  当最大李雅普诺夫指数 $MLE > 0$（系统处于高度混沌的蝴蝶效应边缘）时，极易发生剧烈相变。若模型给出“确信的稳定/低波动”预测，则施加基于 `MLE * ReLU(阈值 - |Pred|)` 的惩罚。

这套损失函数迫使网络在物理指标发出预警时，必须勇敢地指出突破或反转的方向，从而成为一个真正的相变探测器。
