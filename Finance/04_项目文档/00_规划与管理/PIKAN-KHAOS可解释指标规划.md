# PIKAN-KHAOS 可解释指标规划

## 1. 目标与交付
- 解释“经过 PI-KAN 训练后的 KHAOS 模型到底输出了什么信息、这些信息各自代表什么市场含义”。
- 基于模型能力，规划一套“高可读 + 高可解释 + 可验证”的 TradingView 指标表达形式。
- 输出可执行的实现路线：信号层、状态层、解释层、验证层分别如何落地。

## 2. 当前痛点（需要解决）
- 面板信息较多但层次不清，交易决策与模型状态混杂。
- 信号触发条件可见，但“为何触发”缺少直观解释路径。
- 训练输出（force/energy/regime）与用户操作（入场/过滤/风控）映射不明确。
- 缺少统一的“模型能力边界提示”，易误用为单一预测器。

## 3. 先澄清：PI-KAN 训练后 KHAOS 可提供的信息

### 3.1 核心可观测量
- **Force（驱动力）**：由蒸馏后的多项式近似模型输出，反映短期“反转/回归”倾向强弱与方向。
- **EKF Energy（偏离能量）**：价格对状态估计的偏离程度（z-score），衡量“异常冲击”是否足够大。
- **Regime Validity（状态有效性）**：由波动门与结构门共同决定，告诉你当前市场是否允许执行反转信号。
- **Bias/Main Line（势能偏置）**：价格相对状态轨道的位置，辅助判断当前偏空/偏多环境。
- **Composite Score（复合评分）**：将驱动力与能量融合，作为可读性更高的总体判断分数。

### 3.2 能力边界（必须显式告诉用户）
- 它擅长**相变/反转触发筛选**，不是传统趋势跟随器。
- 对低波动、强单边趋势场景会主动降权或屏蔽（Regime Blocked）。
- 信号是“条件满足下的概率优势”，不是确定性预测。

## 4. 指标设计原则（面向可解释性）
- **分层显示**：先看“是否可交易”，再看“方向”，最后看“触发细节”。
- **同一语义同一视觉**：状态类用离散色块，强度类用连续柱/线，信号类用脉冲标记。
- **一屏可回答三问**：现在是什么状态？为什么给信号？信号强度有多高？
- **保留审计线索**：每个信号都可追溯到 Force、Energy、Regime 三项证据。

## 5. 推荐的指标形式（TradingView UI 方案）

### 5.1 主图层（执行层）
- BUY/SELL 脉冲标签（仅在 Regime 有效时允许触发）。
- Top/Bottom Zone 区间线（显示持续有效期，而非单点箭头）。
- 背景状态（可选）：趋势稳定/高混沌/中性。

### 5.2 副图层（解释层）
- **Histogram 1：Energy**（异常程度，强调“是否值得关注”）。
- **Line 1：Main Bias**（当前偏置方向）。
- **Line 2：Composite**（综合强度，作为人眼快速判读主线）。

### 5.3 状态面板（摘要层）
- 固定显示：State / Main / Force / Energy / Composite / RegimeScore。
- 颜色语义：绿=可执行，红=屏蔽，蓝=偏多，橙=偏空。

### 5.4 参数分组（交互层）
- Physics Core：alpha、hurst_fast、hurst_slow。
- Regime Analysis：pe_len 等结构参数。
- Signal Logic：sigma_trig、sigma_exit、gates。
- Display：显示开关与平滑参数（不影响核心决策逻辑）。

## 6. 实施步骤（执行阶段）
1. **信息架构重排**：明确主图/副图/状态面板职责，冻结视觉语义表。  
2. **变量到语义映射固化**：为 Force/Energy/Regime/Composite 定义统一解释文本。  
3. **信号解释链落地**：每个入场点满足“门控通过 + 驱动力阈值 + 能量确认”。  
4. **显示参数与决策参数解耦**：Display 仅影响可视化，不改变信号核心。  
5. **验证基线建立**：本地脚本检查 Pine 参数兼容 + Python/Pine 数值一致性。  
6. **人工验收模板输出**：按资产/周期记录胜率、回撤、信号密度、误报类型。  

## 7. 验证计划（你手动执行）
- **一致性验证**：先跑本地 Pine 校验脚本，确保静态语法与数值一致性通过。
- **图表验证**：BTC/EURUSD/Gold 各选 1h 与 4h，检查状态-信号是否一致。
- **行为验证**：在高波动与单边行情对比 Regime 是否正确屏蔽误触发。
- **结果记录**：按“信号质量、可解释性、可操作性”三维打分。

## 8. 预期产出（执行完成后）
- 一份“可读性优先”的 KHAOS Pine 指标版本（含清晰分层与状态表）。
- 一份“模型能力说明卡”（告诉使用者可做什么、不能做什么）。
- 一份“手动验证记录模板”（支持迭代调参与版本对比）。
