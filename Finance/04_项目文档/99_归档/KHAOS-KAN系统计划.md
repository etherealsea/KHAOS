# 基于物理信息增强神经网络 (PI-KAN) 的金融时间序列相变探测研究 —— 系统设计文档  


## 一、设计目标与原则

本系统以 KHAOS 核心物理模型（Extended Kalman Filter + Hurst-driven damping + Permutation Entropy）作为前置物理传感器层，结合 Kolmogorov-Arnold Network（KAN）作为可训练非线性映射模块，构建一个完整的物理信息神经网络（Physics-Informed Neural Network），最终输出可直接用于量化交易策略的连续或离散信号。

设计遵循以下原则：  
1. 物理层数学结构严格保持原始文档定义，不做任何简化或替代  
2. 全部计算流程端到端可微  
3. 神经网络部分仅使用 KAN 架构，不引入注意力机制、RNN、Transformer 等序列建模组件  
4. 系统整体参数量可控、可解释性优先  
5. 最终输出为单资产、单周期的确定性交易信号，便于与传统技术指标策略直接对比

## 二、系统整体架构

```
输入：OHLCV 时间序列 (T × 5) 或批次 (B × T × 5)
       ↓
KHAOS 物理传感器层（完全可微实现）
       ↓
输出 5 维物理状态向量 Ψ_t = [H_t, E_vol, M_kin, A_res, S_ent]^T
       ↓
与少量原始统计特征拼接 → X_t ∈ ℝ^{T × D}，D ≤ 15
       ↓
KAN 主干网络（1~3 层）
       ↓
标量输出：风险标准化未来收益预测 ŷ_t
       ↓
交易信号生成（tanh 压缩或分段映射）
```

## 三、KHAOS 物理传感器层（可微实现要求）

| 模块                     | 原始公式保持不变 | 可微实现要点                              |
|--------------------------|------------------|-------------------------------------------|
| Extended Kalman Filter   | 状态 [p, v]，非线性转移 ρ(H) | 手写 predict/update 步骤，使用 torch.autograd |
| Damping function ρ(H)    | ρ(H) = 0.5 + 0.5/(1+e^{-10(H-0.5)}) | 直接实现                                  |
| Hurst 快速代理           | H ≈ 0.5 + 0.3 × tanh(m/10)         | 最小二乘闭式解或 torch.linalg.lstsq       |
| 自适应 Q/R               | Q/R ∝∝ max(1, 100σ_vol)          | 可微 max 近似或 torch.clamp               |
| Permutation Entropy      | 标准定义（order=3~5, τ=1）         | 先使用 hard 统计 + straight-through，后续可换 soft 版本 |
| 最终 5 维特征             | 完全按原文档公式                     | 所有中间变量保留以便构造物理约束损失      |

## 四、KAN 主干网络设计

### 4.1 输入特征组成（推荐配置）
| 特征来源           | 维度 | 说明                              |
|--------------------|------|-----------------------------------|
| KHAOS 物理状态 Ψ_t | 5    | 核心物理特征                      |
| 价格回报率序列     | 3    | ret_1, ret_3, ret_5               |
| 成交量比率         | 2    | vol/vol_ma20, vol/vol_ma60        |
| 高低价偏离         | 2    | (high-low)/close, (close-open)/close |
| 总计               | ≤ 15 | 可根据实际效果删减                |

### 4.2 KAN 网络拓扑（推荐搜索空间）
| 层数 | 隐藏宽度选项       | grid_size | spline_order | 备注                  |
|------|--------------------|-----------|--------------|-----------------------|
| 1    | [8, 12, 16]        | 3~8       | 3            | 起点配置              |
| 2    | [12, 8], [16, 12]  | 5~10      | 3            | 第二阶段可选          |
| 3    | [16, 12, 8]        | 5~8       | 3            | 仅在前述配置稳定后使用 |

输出层始终为 1 维标量（不使用激活函数）。

### 4.3 KAN 特有超参数
- 样条正则：L1 penalty on spline coefficients，推荐 1e-5 ~ 1e-4  
- 基础激活函数：SiLU 或 Softplus  
- 网格更新策略：训练中可开启 grid update（每 20 epoch）

## 五、训练目标与标签定义

### 主任务
预测未来 h 根 K 线的风险标准化收益：
```
y_t^h = (P_{t+h} / P_t - 1) / σ_t
```
其中 σ_t 为前 20~40 根 K 线的波动率估计（ATR 或滚动标准差）  
推荐 h ∈ {4, 8, 16}，可单任务或多任务并行

### 主要损失函数
- Huber Loss（δ=1.0）或 MSE（早期实验可用）

### 辅助任务（可选）
1. 未来波动率预测  
2. 体制标签预测（Hurst>0.65 → 趋势）

## 六、物理信息约束损失（Physics-Informed 核心）

所有约束均在最后若干 timestep 上计算，确保可微：

| 编号 | 约束名称               | 公式（简化）                                    | 物理意义                     |
|------|------------------------|------------------------------------------------|------------------------------|
| P1   | 动能方向一致性         | ReLU(- ŷ_t × M_kin)                            | 正动能时不应预测下跌         |
| P2   | 趋势行情动量守恒       | when H>0.65: MSE(ŷ_t - ŷ_{t-1})                 | 趋势中预测不应剧烈反转       |
| P3   | 高熵抑制过度自信       | S_ent × |ŷ_t|                                   | 市场混乱时预测幅度应收敛     |
| P4   | 异常事件放大预测波动   | ReLU(threshold - |ŷ_t|) × |A_res|               | 残差大时不能预测太平滑       |
| P5   | 收益与 EKF 速度同号    | ReLU(- ŷ_t × v_ekf)                            | 最强的物理一致性约束         |

约束权重 λ 从小到大逐步加入，典型范围 0.01 ~ 1.0

## 七、信号与策略生成

训练完成后，固定模型参数，推理时仅使用最后一根的有效预测：

```
raw_pred = model(last_T_timesteps)[-1]
signal   = tanh(k × raw_pred)          # k ∈ [1, 5] 可调
# 或分段仓位
position =  1    if raw_pred >  threshold1
           -1    if raw_pred < -threshold1
            0    otherwise
```

最终输出的 position 序列直接用于回测，与传统均线、布林带、RSI、MACD、唐安奇通道等策略进行并列比较。

## 八、实验与验证要求

1. 所有训练、验证、测试集严格按时间划分，禁止未来数据污染  
2. 特征标准化（如有）仅使用训练集统计量  
3. 每完成一个阶段（新增约束、加深 KAN 等）均保存完整模型与预测序列  
4. 最终效果评估指标包括但不限于：年化收益、最大回撤、Calmar 比率、胜率、RankIC 等  
5. 与传统技术指标策略的对比在完全相同的数据、成本、滑点假设下进行

本设计文档定义了从物理建模到最终可执行交易信号的完整技术路径，确保物理约束与数据驱动学习在同一框架内协同工作。后续所有实现均应严格遵循本规范。
