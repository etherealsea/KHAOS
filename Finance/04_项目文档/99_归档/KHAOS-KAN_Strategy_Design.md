# KHAOS-KAN Swing-Surge 策略设计方案

## 1. 核心理念 (Core Philosophy)
本策略名为 **Swing-Surge (波段-涌浪)**，旨在结合 **KHAOS (混沌动力学系统)** 的分形感知能力与 **KAN (Kolmogorov-Arnold Networks)** 的非线性拟合能力。
策略的核心逻辑是：**在分形有序时顺势而为，在分形极值时确信反转。**

*   **KHAOS** 负责感知市场的“能量”与“体制”。
*   **EMA20** 负责界定物理世界的“惯性方向”。
*   **ATR** 负责量化“空间”与“风险”。
*   **KAN** 作为大脑，负责学习上述因子之间的非线性关系，决定最佳的“开火时机”。

## 2. 策略工具箱 (Toolbox)
严格遵守约束，仅使用以下工具：
1.  **KHAOS System**: 包含 `KHAOS Oscillator` (能量), `Hurst` (分形维数/趋势强度)。
2.  **EMA20**: 20周期指数移动平均线，作为短期趋势的生命线。
3.  **ATR**: 平均真实波幅，用于动态止盈止损与波动率归一化。
4.  **KAN Model**: 决策核心。

## 3. 特征工程 (Feature Engineering)
我们将为 KAN 提供 6 个经过归一化的核心特征，使其能够感知市场全貌：

| 编号 | 特征名称 | 描述 | 物理含义 |
| :--- | :--- | :--- | :--- |
| 1 | **KHAOS_Val** | KHAOS 指标原值 | **能量位置**：当前是超买还是超卖？动能是正还是负？ |
| 2 | **KHAOS_Vel** | `KHAOS - KHAOS[1]` | **能量加速度**：动能是在增强还是衰减？ |
| 3 | **Hurst_Exp** | Hurst 指数 (0-1) | **市场体制**：`>0.6` 为强趋势，`<0.45` 为强均值回归。 |
| 4 | **Trend_Pos** | `(Close - EMA20) / ATR` | **偏离度**：价格距离趋势线有多远？(归一化距离) |
| 5 | **Trend_Slope** | `(EMA20 - EMA20[1]) / ATR` | **趋势强度**：趋势线的倾斜角度有多陡峭？ |
| 6 | **Vol_State** | `ATR / SMA(ATR, 50)` | **波动状态**：波动率是在压缩(蓄势)还是在爆发(高潮)？ |

## 4. 交易逻辑 (Trading Logic)

### 4.1 顺势波段模式 (Trend Swing Mode) - *主策略*
*   **场景**: 市场处于有序趋势中。
*   **逻辑**:
    *   **方向确认**: 价格位于 EMA20 之上 (做多) 或之下 (做空)。
    *   **结构确认**: Hurst 指数较高 (例如 > 0.55)，表明存在持续性。
    *   **触发信号**: KHAOS 能量从负转正 (零轴突破) 或在回调后再次发散。
*   **KAN 的任务**: 学习在 Trend_Slope 和 Hurst 的不同组合下，KHAOS 需要达到多少阈值才算是有效突破，过滤掉假信号。

### 4.2 确定性反转模式 (High-Confidence Reversal) - *副策略*
*   **场景**: 市场过度延伸，能量耗尽。
*   **逻辑**:
    *   **极端偏离**: Trend_Pos 达到极端值 (如 > 3.0 或 < -3.0)。
    *   **能量衰竭**: Hurst 指数极低 (< 0.45)，表明趋势结构瓦解，转为随机游走或反转。
    *   **触发信号**: KHAOS 出现背离或极端值回归。
*   **KAN 的任务**: 严格区分“主升浪中的回调”与“真正的趋势终结”。

## 5. 动态风控体系 (Dynamic Risk Management)

### 5.1 动态止损 (Dynamic Stop Loss)
放弃固定点数止损，采用基于 ATR 的“呼吸止损”：
*   **公式**: `Stop_Price = Entry_Price - (k_SL * ATR)`
*   **机制**:
    *   **初始止损**: `k_SL` 通常为 2.0 ~ 3.0。
    *   **追踪止损**: 当盈利超过 1R 时，止损上移至 `Entry_Price` (保本)。随后沿 EMA20 或 ATR 轨道追踪。

### 5.2 动态止盈 (Dynamic Take Profit)
*   **公式**: `Target_Price = Entry_Price + (k_TP * ATR)`
*   **目标盈亏比**:
    *   基础设定: **R:R >= 1:2**。
    *   KAN 优化: KAN 可以根据波动状态 (Vol_State) 动态输出推荐的盈亏比。例如在低波动爆发期，追求 1:5；在高波动震荡期，满足于 1:1.5。

## 6. KAN 学习目标 (Training Objective)
*   **输入**: 上述 6 个特征的滑动窗口 (例如过去 30 根 K 线)。
*   **输出**: 
    *   **Signal (信号强度)**: (-1 到 1)。> 0.6 做多，< -0.6 做空。
    *   **Confidence (置信度)**: (0 到 1)。用于动态调整仓位大小。
*   **损失函数 (Loss Function)**:
    *   不仅最大化收益率，还要引入 **Sortino Ratio** 作为奖励，惩罚下行波动。
    *   引入 **交易频率惩罚**，迫使 KAN 仅在确定性最高时出手，避免过度交易。
